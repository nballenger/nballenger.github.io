Chapter Summaries

Chapter 1: Testing for Fun and Profit

  • Unit Testing — A unit is a functionality that cannot be subdivided, unit tests individually test each unit within the software.
  • Integration Testing — Test interactions between units, though each test is run in isolation.
  • System Testing — Test portions of the system with all pieces intermeshed.

Chapter 2: Doctest: The Easiest Testing Tool

  • Doctest takes the output of an interactive session and uses it to test.
  • A sample test.txt file: 
    This is a doctest for arithmetic.
>>> 2 + 2
4
>>> 3 * 3
10
  • Running that test: python -m doctest test.txt

Chapter 4: Breaking Tight Coupling by using Mock Objects

  • Chapter uses the Python Mocker toolkit, which is not part of the standard install.
  • Mock objects pretend to do what a real object would do, to isolate units that interact with that object.
